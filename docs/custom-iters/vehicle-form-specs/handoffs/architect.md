## Risk Register
1. **Widget file split (med)**: `vehicle_form.dart` is a large monolithic file that violates 1-widget-per-file. The rewrite risk is a UI regression in an existing working form. Mitigation: keep section-by-section parity and test create + edit flow manually after.
2. **Nav header change (med)**: Replacing `AppAppBar` with Pencil-style "Cancelar | Title | Guardar" nav. The current back/close behavior changes. Mitigation: "Cancelar" → `context.pop()`, "Guardar" → `_saveVehicle()`.
3. **build_runner regeneration (low)**: After adding fields to `VehicleModel`, `VehicleDto`, the generated `.g.dart` and `.freezed.dart` must be regenerated. If any other model references the old shape there could be type errors. Mitigation: run `dart run build_runner build --delete-conflicting-outputs` and fix any errors.
4. **`_vehicleRequest` manual map (low)**: This map in `vehicle_repository_impl.dart` manually lists every field. New spec fields will NOT be sent unless explicitly added. This is a known footgun — must add all 4 fields.
5. **Prisma migration in non-CI environment (low)**: Backend agent runs migration locally. No prod impact. Migration is additive-only.
