## Changes summary

### lib/main.dart
Replaced the conditional token guard (`if (mapboxToken != null && mapboxToken.isNotEmpty)`) with a fail-fast `assert` followed by an unconditional `MapboxOptions.setAccessToken(mapboxToken!)`. This makes a missing/empty token immediately visible as an assertion failure in debug builds rather than silently skipping initialization.

### ios/Runner/Info.plist
Added `<key>MBXAccessToken</key>` with the literal `pk.*` token value (matching `MAPBOX_ACCESS_TOKEN` from `android/local.properties` line 7). The key is placed after the `#endif` closing the dev ATS block and before the closing `</dict>` tag. This ensures the Mapbox iOS SDK (which initializes natively before the Dart isolate starts) has a token available at startup. The build-variable form `$(MAPBOX_PUBLIC_TOKEN)` was deliberately NOT used because no xcconfig defines that variable.

### lib/shared/widgets/map/route_map_preview.dart
Added `bool _mapLoadError = false;` field to `_RouteMapPreviewState`. Changed the `MapWidget` show condition from `if (_hasCoordsToShow)` to `if (_hasCoordsToShow && !_mapLoadError)`. Added `onMapLoadErrorListener` to the `MapWidget` constructor: when triggered it calls `setState(() => _mapLoadError = true)`, which causes the map surface to be replaced by the existing "Vista previa del mapa" placeholder (map icon + subtitle text). The existing `hide Error` import is preserved unchanged.

### lib/features/events/presentation/tracking/widgets/live_map_widget.dart
Added optional `final ValueChanged<String>? onMapError;` field to `LiveMapWidget` and `this.onMapError,` to the constructor. Added `onMapLoadErrorListener` to the `MapWidget` in `build()` that calls `widget.onMapError?.call(data.message)`, passing only the message string upward and not leaking `MapLoadingErrorEventData` to consumers.

### lib/features/events/presentation/tracking/widgets/live_map_body.dart
Wired `onMapError` on the `LiveMapWidget` instantiation. On error it gets the `ScaffoldMessenger`, removes any current snack bar (to prevent stacking on repeated listener invocations), and shows a `SnackBar` with `context.l10n.map_loadError`.

### lib/l10n/app_es.arb + regenerated files
Added `map_loadError` key with value `"No se pudo cargar el mapa."` and its `@map_loadError` metadata annotation, placed after the existing `map_geocodeError` key. Ran `flutter gen-l10n` which regenerated `app_localizations.dart` and `app_localizations_es.dart`. Both files now expose `String get map_loadError`.
