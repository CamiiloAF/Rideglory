## Domain context

Rideglory is a Flutter mobile app. QA role: write and verify tests; confirm acceptance criteria are met; produce the test catalog. No Playwright — Flutter test tooling only.

**Personas under test:**
- **Rider** — all browsing, registration, garage, tracking, profile flows
- **Event Organizer** — event creation, registration approval/rejection, attendee list, live tracking map

---

## Scope boundaries

QA works on:
- Unit tests: `test/features/<feature>/domain/` and `test/features/<feature>/data/`
- Widget tests: `test/features/<feature>/presentation/`
- Integration tests: `integration_test/` (happy path per feature, real dev backend)
- `dart analyze` run as final gate before iteration close

QA does NOT:
- Write backend tests (that's the backend agent's responsibility)
- Run Playwright or web tests
- Test on real hardware (emulator is sufficient for CI)

---

## Key acceptance patterns

### Definition of done (every iteration)
1. `dart analyze` passes with **zero** violations.
2. `flutter test` passes **100% green**.
3. No hardcoded Spanish strings in UI.
4. No layer violations (domain: no Flutter; presentation: no HTTP).
5. No raw Material widgets where a shared equivalent exists.

### Test structure and tooling
- **Mock library:** `mocktail` — `class MockFoo extends Mock implements FooRepository`
- **Cubit testing:** `bloc_test` — `blocTest<MyCubit, ResultState<T>>(...)`
- **Network images in widget tests:** wrap with `mockNetworkImages()` from `network_image_mock`
- **Widget test pattern:** `BlocProvider<CubitType>.value(value: mockCubit)` wrapping the page under test; use `whenListen` to emit state sequences
- **Integration tests:** real dev backend at localhost; real Firebase Auth token; no mocks

### Mandatory test cases per cubit (blocTest groups)
1. `initial` state is emitted on creation
2. `loading` emitted when async method called
3. `data` emitted on success (with expected model value)
4. `empty` emitted when result is empty list
5. `error` emitted when use case returns `Left(DomainException)`

### Mandatory widget test cases per page
1. Loading skeleton renders (shimmer visible, no data widgets)
2. Data state renders expected content (title/name/card visible)
3. Empty state renders `EmptyStateWidget` (no crash, no blank screen)
4. Error state renders error banner with retry button

### Integration test structure
```dart
group('Feature happy path', () {
  testWidgets('complete flow', (tester) async {
    // auth token injection → API call → state update → UI render
  });
});
```

## Edge cases to test

- Network timeout on API call → error state → retry button → success state
- Firebase token expiry mid-session → interceptor refreshes token → request retried transparently
- PDF upload with no extractable date → AI returns null → manual entry fallback renders automatically
- Unsplash API rate limit (503) → error banner in event form → no crash
- SOS triggered by rider not on map → overlay still shows with last known position
- FCM notification tap when app is terminated AND user is not yet logged in → auth redirect → then deep-link
