## Domain context

Rideglory is a Flutter mobile app (Android + iOS) for motorcycle riders and event organizers. The frontend agent implements features in `lib/` following Clean Architecture (domain / data / presentation per feature), BLoC/Cubit state management, and the rideglory-coding-standards.

**Personas in UI terms:**
- **Rider** — taps to browse events, register, see their garage, track live rides, view profiles. Glove-friendly touch targets.
- **Event Organizer** — uses the same app with organizer-gated flows (create event, approve/reject registrations, view attendee list).

All UI copy in Spanish. All user-visible strings in `lib/l10n/app_es.arb` via `context.l10n.<key>`. Key prefix per feature (e.g., `profile_`, `vehicle_`, `event_`, `soat_`, `sos_`).

## Key acceptance patterns

Every iteration, without exception:
1. `dart analyze` — zero violations.
2. `flutter test` — 100% green.
3. `dart run build_runner build --delete-conflicting-outputs` — no conflicts.
4. No hardcoded Spanish strings in UI — all via `context.l10n.<key>`.
5. No raw Material widgets where a shared equivalent exists.
6. No layer violations: domain files have no Flutter imports; data files have no `BuildContext`.
7. Every new async feature has a loading skeleton, empty state (`EmptyStateWidget`), and error banner with retry.

**ResultState<T> pattern (mandatory for all async):**
```dart
@freezed
class ResultState<T> {
  const factory ResultState.initial() = Initial<T>;
  const factory ResultState.loading() = Loading<T>;
  const factory ResultState.data({required T data}) = Data<T>;
  const factory ResultState.empty() = Empty<T>;
  const factory ResultState.error({required DomainException error}) = Error<T>;
}
```
Never use boolean loading/error flags. Never expose DTOs in presentation layer.

**Cubit pattern:**
- Simple single-result: `Cubit<ResultState<T>>`
- Multiple independent results: `@freezed` state class with one `ResultState<T>` field per result

**DI:** New services/repositories: `@injectable` or `@lazySingleton`. New global cubits: `@lazySingleton` + added to root `MultiBlocProvider` in `main.dart`.

## Complex flow notes (high-risk iterations)

### SOAT upload (Iter 3b)
Six distinct UI states — each is a separate widget/state, not a single spinner:
1. File picker trigger (button in vehicle detail)
2. Upload progress indicator (Firebase Storage upload %)
3. AI extraction loading ("Extrayendo fecha con IA…")
4. Confirmation form (show extracted date, editable, low-confidence warning if applicable)
5. Manual entry fallback (date picker if AI fails or user rejects)
6. Success → return to vehicle detail with updated badge

### FCM deep-link routing (Iter 6a)
- FCM payloads carry scalar IDs only (`{ route: 'registrationDetail', registrationId: '...' }`)
- `NotificationNavigator` handles routing after app hydration — avoids go_router `extra` non-serializability
- Must handle 3 app states: foreground (in-app banner), background (tap notification), terminated (tap notification → cold start → navigate)
- Implement "by-id" route variants for all 5 target screens

### SOS overlay (Iter 6b)
- Top-anchor banner stack — does NOT cover the map
- Multiple simultaneous SOS: `List<SosAlertModel>`, each shown as a stacked banner
- SOS sender can cancel (sends `{ type: "sos", active: false }`) — overlay removes for that riderId
- No organizer dismiss in v1

## Iteration 1 learnings (2026-05-12)

### build_runner / mockito conflict
`network_image_mock: ^2.1.1` pulls in `mockito 5.6.5`. The project overrides `analyzer: ^8.0.0` which breaks the mockito builder compilation inside build_runner. The DI config (`injection.config.dart`) must be manually patched when adding new `@injectable`/`@lazySingleton` classes until this conflict is resolved. Use import alias `_i<number>` matching existing patterns; pick numbers > 2000 to avoid collisions with future generated output.

### Manual DI pattern
When adding `@lazySingleton` classes that can't be auto-generated:
1. Add import with unique alias (`_i2001`, `_i2002`, ...) before the `GetItInjectableX` extension.
2. Register with `gh.lazySingleton<_iXXX.TheClass>(() => _iXXX.TheClass(gh<_iYYY.Dependency>()))` before `return this`.
3. Add `@injectable` use cases as `gh.factory<...>`.

### l10n without build_runner
Run `flutter gen-l10n` as a standalone command (not via build_runner) to regenerate localization files. The `generate: true` in pubspec.yaml / flutter section makes this work independently.

### ProfileCubit scope
`@lazySingleton` is correct for `ProfileCubit` — it persists profile data across tab switches. Call `reset()` on logout to clear state. Place in root `MultiBlocProvider` in `main.dart` so it's available inside and outside the shell route.

### Initials helper
`initialsFromName(String?)` is in `lib/core/utils/initials.dart`. The existing `Initials.buildFromFullName` static method also exists — prefer the top-level function for parity with the architect spec and future attendee list usage.
