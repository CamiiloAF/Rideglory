## Domain context

The backend is **rideglory-api** at `/Users/cami/Developer/Personal/rideglory-api` — a NestJS microservices system with an api-gateway and independent microservices (events-ms, vehicles-ms, users-ms, maintenances-ms). All HTTP routes are protected by Firebase Auth guard; the gateway validates Firebase ID tokens and forwards requests to microservices via a message bus.

**Domain business rules:**
- A rider must be authenticated to access any resource (no public endpoints except auth).
- An event organizer must own the event to approve/reject registrations, change event status, or delete it.
- Vehicle documents (SOAT, future: tecno) are associated to a `(userId, vehicleId)` pair.
- FCM device tokens are associated to `userId` in a `device_tokens` table; one user may have multiple tokens (multiple devices). Send to all tokens for that user.
- SOS alerts are broadcast to all WebSocket clients in the same `eventId` session — not via FCM.
- Event status transitions drive FCM sends: `pending → in_progress` notifies all approved registrants; `any → cancelled` notifies all registrants of any status.

## Key acceptance patterns

Every backend change must:
1. Pass `nest build` with zero TypeScript errors.
2. All protected routes have `@UseGuards(FirebaseAuthGuard)` — no endpoint exposed unauthenticated.
3. Firebase Storage PDF access uses Admin SDK `bucket.file(path).download()` — never a public URL.
4. FCM payloads: scalar IDs only (`eventId`, `registrationId`, `route`) — no nested objects.
5. Claude API calls use `claude-haiku-4-5` model (cost-efficient for extraction tasks).
6. New Prisma models must have a migration file; never modify prod schema without a migration.

**API contract conventions:**
- `POST` bodies: validated with NestJS `class-validator` DTOs.
- Error responses: `{ message: string, statusCode: number, error: string }`.
- Firebase Storage paths passed TO backend: `insurance/{userId}/{vehicleId}/soat.pdf`.
- Backend returns `storageUrl` (download URL with short TTL) for frontend display.

## New API contracts (from plan)

### SOAT (Iter 3a)
```
POST /vehicles/:vehicleId/insurance
Body: { docType: 'soat' | 'tecno' | 'other', storagePath: string }
Response: { id, vehicleId, docType, storagePath, expirationDate, extractionConfidence, status }

GET /vehicles/:vehicleId/insurance
Response: InsuranceDocument[]

PATCH /vehicles/:vehicleId/insurance/:id
Body: { expirationDate: string } // manual override
Response: InsuranceDocument
```

### AI Cover (Iter 4)
```
POST /events/generate-cover
Body: { title: string, type: string, location: string }
Response: { imageUrl: string, query: string }
```

### Recommendations (Iter 5)
```
GET /events/recommendations
Headers: Authorization: Bearer <firebase-id-token>
Response: EventDto[] (max 5)
```

### FCM Token (Iter 6a)
```
POST /users/device-token
Body: { token: string, platform: 'android' | 'ios' }
Response: { success: boolean }
```

### SOS WebSocket message (Iter 6b)
```json
{ "type": "sos", "riderId": "...", "riderName": "...", "lat": 0.0, "lng": 0.0, "eventId": "...", "active": true }
```
